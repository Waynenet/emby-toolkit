# 处理流程

本页以主动入库、Webhook 辅助事件、任务链和实时监控四条主线描述处理流程。

## 1. 主动入库流程

触发入口：实时监控（`monitor_service.py`）

- 识别新增 STRM 或实体媒体，将首次刮削元数据和路径身份写入 `media_metadata`。
- 主动通知 Emby 扫描文件，优先由桥接插件上报 Item ID，超时后再按路径轮询兜底。
- 将取得的 Item ID 交给现有核心处理流程，完成元数据补全、翻译、数据库写入与 Emby 回写。
- 根据处理结果刷新追剧、合集、封面并发送入库或追更通知。

## 2. Webhook 辅助事件

触发入口：`/webhook/emby`（`routes/webhook.py`）

- 不处理 Emby 的 `item.add`、`library.new` 事件，也不再通过 Webhook 触发音乐入库媒体信息刷新。
- 继续处理播放、用户状态、元数据、图像、深度删除和合集成员删除等事件。
- 同一入口继续接收 MoviePilot 订阅、下载和整理事件。

## 3. 任务链流程

任务链由 `tasks/core.py` 注册，配置项定义于 `config.ini`（Scheduler 部分）。

- **高频任务链**：用于日常刷新的核心任务序列。
- **低频任务链**：用于维护、清理与低频场景。
- **任务执行**：任务队列按顺序执行，每个任务更新进度与日志。

典型任务：

- 同步演员数据、演员中文化、角色名翻译
- 同步媒体元数据与评分
- 刷新追剧、演员订阅
- 生成封面、刷新合集

## 4. 实时监控流程

监控服务在启用后监听指定目录（`Monitor` 配置）：

- 新增/移动：识别媒体、准备数据库元数据和媒体信息，并进入主动 Item ID 获取流程。
- 删除：对删除事件进行批处理并刷新 Emby 视图。
- 排除路径：命中排除路径的文件仅刷新，不做刮削。

## 5. 反向代理虚拟库

当启用反向代理后：

- 代理层汇总原生库与自建合集；
- 对虚拟库 Items 做排序与分页，必要时进行内存排序回退；
- 通过 Nginx 转发与静态资源处理提供访问入口。
